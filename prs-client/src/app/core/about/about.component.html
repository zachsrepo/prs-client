The Purchase Request System (PRS) is the capstone for the MAXimum boot camp and its completion is required for graduation.
The back end system is comprised of a Microsoft SQL server database and a C# restful API controller.

The database contains five table: User, Vendor, Product, Request, and Requestline.

The User table provides two functions:

  • Provides authentication for users wishing to log in and use the PRS
  • Provides ownership for Requests created (who created and 'owns' them)

The Product table provides two functions:

  • Provides the list of products that can be added to a RequestLine
  • Provides the name and price of products

The Request table is the heart of the PRS. Together with a collection of RequestLine rows makes up the user's request of products. It has xxx functions:

  • Provides the grouping for all the products being requested.
  • Provides the user that created the request
  • Provides the total amount of all the products being requested

The RequestLine table implements a many-to-many relationship between the request and product tables.
Each request can have multiple requestLines and every product can have multiple lines.
This means that every request can be related to multiple products and each product can be related to multiple requests.

The RequestLine holds all the product for a particular request.
